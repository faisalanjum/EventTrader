Learning Loop — How It Works

Two types of learning, one flow. The learner writes files. Consumers read them. That's the whole system.

Two Types of Learning

Type 1: In-Context Learning (LLM)

What: The prediction LLM sees free-form text about its past performance before making the next prediction.

How: Orchestrator loads learner files → injects into prediction bundle as context.

Example: "Last time you predicted AAPL +5-8%, actual was +7.2%. You assumed guidance was raised but it was merely maintained."

Effect: LLM naturally recalibrates — like a human reading their past mistakes before making a decision.

No minimum sample needed. Even 1 past trade is useful context. The LLM decides how much weight to give it.
Type 2: Parameter Learning (Daemon Config)

What: Numeric config values (stop distance, hold time) auto-adjust based on trade outcomes.

How: Daemon reads auto_tune from global.json at startup → applies within hard bounds.

Example: "17% of stops fired on noise at ATR×2.0. Recommendation: ATR×2.3"

Effect: Stops get slightly wider → fewer noise stop-outs → more winners held.

Minimum 30 trades required. Config changes need statistical evidence. One data point doesn't change anything.

What the Learner Writes

File 1: global.json — cross-ticker patterns + config

{ "schema_version": "v1", "updated_at": "2026-04-08T15:30:00Z", "trades_analyzed": 47, "historical_trades": 35, "live_trades": 12, // FREE-FORM TEXT — LLM reads this before every prediction "prediction_lessons": "Tech sector: you overestimate magnitude by ~3pts when guidance is maintained not raised (n=18). EPS surprise >10% is your strongest signal at 82% accuracy (n=11). Confidence 75-80 actually wins 68%, not 75-80% (n=23).", // FREE-FORM TEXT — LLM reads this before every reassessment "reassessment_lessons": "Your confirmed calls are reliable at 85% (n=12). Transcript reassessments more valuable than Benzinga news reassessments.", // STRUCTURED — daemon applies these automatically within bounds "auto_tune": { "ATR_MULTIPLIER": 2.3, "GAP_FADE_PCT": 55 }, // STRUCTURED — daemon logs these, human reviews later "candidates": { "CONFIDENCE_THRESHOLD": { "value": 72, "reason": "5 skipped trades at 72-74 averaged +6.1%" } } }

File 2: ticker/AAPL.json — per-ticker history

{ "schema_version": "v1", "updated_at": "2026-04-08T15:30:00Z", "ticker": "AAPL", "trades_analyzed": 3, // FREE-FORM TEXT — LLM reads this when predicting AAPL specifically "lessons": "Q1 FY2026: predicted +5-8% conf 82, actual +7.2%. Direction correct but assumed guidance raised when it was maintained — magnitude overestimated. Q4 FY2025: predicted +3-5% conf 71, actual -2.1%. Direction wrong — missed services revenue deceleration. Summary: 2/3 winners. Weight revenue composition more when services >25% of total." }
Key rule: Lessons are free-form text. The learner writes whatever it thinks is useful. No rigid schema — the LLM interprets it naturally. Evidence strength stays in prose: (n=18) or tentative from one case.

Who Reads What, When

Flow 1: Prediction (in-context learning)

8-K filing detected → orchestrator starts

Orchestrator builds the prediction data bundle (8-K text, financials, consensus, etc.)

Orchestrator loads learner files

Reads global.json → extracts prediction_lessons

Reads ticker/AAPL.json → extracts lessons

Adds both to the prediction bundle as context

Prediction LLM receives bundle + lessons

The LLM sees the 8-K data AND its past track record for AAPL AND global patterns.

It naturally adjusts: "last time I overestimated AAPL when guidance was maintained..."

LLM outputs prediction

direction, confidence, expected_move_range — now informed by past mistakes

Flow 2: Reassessment (in-context learning)

Transcript or Benzinga news arrives during open trade

Trade daemon detects the event

Daemon loads learner files

Reads global.json → extracts reassessment_lessons

Reads ticker/AAPL.json → extracts lessons

Includes in the reassessment message alongside the new evidence

LLM reassesses thesis with lessons

Sees: new evidence + original thesis + trade state + "your confirmed calls are 85% reliable"

Outputs: confirmed / unchanged / invalidated

Daemon acts deterministically

confirmed → HOLD. invalidated → EXIT. LLM failure → no_change.

Flow 3: Daemon startup (parameter learning)

Daemon starts (or restarts)

Loads config file with default values (ATR_MULTIPLIER=2.0, etc.)

Reads global.json auto_tune

Finds: "ATR_MULTIPLIER": 2.3

Checks: trades_analyzed ≥ 30? Yes (47). Within bounds [1.5, 3.0]? Yes. Apply.

Config updated for this session

ATR_MULTIPLIER is now 2.3 instead of 2.0 for all NEW trades this session

Open trades keep their original values (frozen in plan.json at entry)

Reads global.json candidates

Finds: "CONFIDENCE_THRESHOLD": 72

Logs: "Candidate: CONFIDENCE_THRESHOLD → 72 (reason: 5 skipped trades at 72-74 averaged +6.1%)"

Does NOT apply. Human reviews later.

Three Tiers — What Can Auto-Adjust

Auto (5 params)

These manage HOW a trade runs. Observable from price action. Don't change which trades you take.

ATR_MULTIPLIER [1.5 — 3.0] MIN_STOP_PCT [2.0 — 5.0] MAX_STOP_PCT [10 — 20] MAX_HOLD_SESSIONS [1 — 3] GAP_FADE_PCT [30 — 70]

Applied automatically at daemon startup if ≥30 trades and within bounds

Candidate (8 params)

These change WHICH trades you take or have noisy counterfactuals. Feedback loop risk.

CONFIDENCE_THRESHOLD [60 — 90] MAGNITUDE_THRESHOLD [1.0 — 5.0] SPREAD_EDGE_RATIO [0.15 — 0.40] BENCHMARK_ATR_MULT [1.0 — 3.0] MACRO_ATR_MULT [1.0 — 3.0] REVERSAL_ATR_MULT [0.3 — 1.0] REVERSAL_MOVE_SHARE [0.15 — 0.50] CHASE_MAX_CONSUMED_PCT[50 — 90]

Logged only — human reviews and promotes when confident

Never (5 params)

Leverage and safety. Auto-adjusting these is the fastest path to blowing up.

RISK_PER_TRADE_PCT ACCOUNT_KILL_SWITCH_PCT MAX_POSITIONS ACCOUNT_MODE DATA_MODE

Human-only — never in global.json, never auto-applied

Why the split?
Auto = "stop was too tight, trade would have won" — clear single-stock counterfactual, low risk.
Candidate = "we skipped a trade that would have won" — changes the population, creates feedback loops.
Never = "trades are going well, increase leverage" — the classic blow-up pattern.

Concrete Example: Trade #48

Setup: AAPL files 8-K at 4:15 PM. 47 prior trades exist. The learner has already analyzed them.

What the prediction LLM sees (in-context learning):

// From global.json → prediction_lessons "Tech sector: you overestimate magnitude by ~3pts when guidance is maintained not raised (n=18). EPS surprise >10% is your strongest signal at 82% accuracy (n=11)." // From ticker/AAPL.json → lessons "3 prior AAPL trades. 2/3 winners. Last time you assumed guidance raised when it was merely maintained — magnitude overestimated. Weight revenue composition when services >25%." // Plus the actual 8-K data, financials, consensus, etc.

The LLM reads this and thinks: "AAPL beat EPS by 12%, guidance language says 'maintained'... last time I overestimated when guidance was maintained. I'll be more conservative on magnitude."

Outputs: direction=long, confidence=78, expected_move_range=[4, 6] instead of [5, 8].

What the daemon uses (parameter learning):

// From global.json → auto_tune (loaded at last daemon startup) "ATR_MULTIPLIER": 2.3 // was 2.0 default, learner widened it "GAP_FADE_PCT": 55 // was 50 default, learner relaxed it

Effect on this trade:

With ATR_MULT=2.0 (old): stop_distance = 2.0 × $4.50 = $9.00 → stop at $176.20 With ATR_MULT=2.3 (learned): stop_distance = 2.3 × $4.50 = $10.35 → stop at $174.85 Result: stop is $1.35 wider. This trade survives a noise dip to $175.50 that would have stopped out the old config. Trade ends +$840 instead of -$490.

Before the Learner Exists (Stage 0)

In-context learning

Orchestrator loads last 5 raw trade/result.json files for this ticker. Passes compact one-line summaries as context.

"AAPL Q1: predicted +5-8%, actual +7.2%, exit=max_hold"

Not as good as learner-distilled lessons, but still useful. The LLM sees raw outcomes.

Parameter learning

No global.json exists → daemon uses default config values.

ATR_MULTIPLIER=2.0, GAP_FADE_PCT=50, etc. — all hardcoded defaults from Section G.

No auto-tuning happens. Manual tuning via config file changes (Block 6C protocol).

Historical PIT runs can bootstrap the learner. Run the prediction pipeline on past earnings (using only data available at filing time). The results feed the learner before a single live trade happens. 30 historical trades → learner can write global.json + ticker files → system starts learning from day 1 of live trading.

One-Page Summary

TRADE CLOSES └─ result.json written (P&L, slippage, exit reason, monitors, reassessments) LEARNER RUNS (separate process, built later) ├─ reads all result.json + skipped.json + reassessments/*.json ├─ analyzes: what went right, what went wrong, what would be better └─ writes: ├─ global.json │ ├─ prediction_lessons → free text, LLM reads │ ├─ reassessment_lessons → free text, LLM reads │ ├─ auto_tune → 5 params, daemon applies within bounds │ └─ candidates → 8 params, daemon logs for human review └─ ticker/AAPL.json └─ lessons → free text, LLM reads for this ticker NEXT TRADE STARTS ├─ Orchestrator loads global.prediction_lessons + ticker lessons │ └─ passes to prediction LLM as context (in-context learning) ├─ Daemon loads global.auto_tune at startup │ └─ applies within bounds if ≥30 trades (parameter learning) LATER, IF REASSESSMENT EVENT FIRES └─ Daemon loads global.reassessment_lessons + ticker lessons └─ includes in reassessment message when transcript/news event arrives (in-context learning) SAFETY ├─ Auto params: bounded [min, max], only 5 stop/horizon params ├─ Candidate params: logged not applied, human promotes ├─ Never params: RISK_PCT, KILL_SWITCH, MAX_POS — human only ├─ Min sample: 30 trades globally for auto-tune └─ Frozen at entry: new learnings affect new trades, never open ones
