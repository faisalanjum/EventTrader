Phase 1 — Trade Execution System

One daemon. One dependency (IBKR + Neo4j for ATR). Three monitoring checks. Everything recorded for learning.

The Flow

Prediction Exists on Disk

prediction/result.json — produced by upstream orchestrator/predictor

▼filesystem scan every 30s

Detect & Adapt

Found result.json without trade/plan.json or trade/skipped.json

Adapt earnings fields → canonical signal: signal_id, symbol, event_id, issued_at, event_time

▼

Gate (5 checks)

direction == long  |  confidence ≥ 75  |  edge ≥ 3%  |  positions < 2  |  capital ok

▼any fail

SKIPPED

trade/skipped.json
with reason + snapshot

▼all pass

FETCH DATA

quote, account, positions,
ATR₁₄, pre-earnings close

▼

Size

stop = clamp(2 × ATR, 3%, 15%)  |  risk = equity × 2%  |  shares = floor(risk / per_share_risk)

If shares < 1 → SKIPPED

▼write plan.json status=entering

Execute: Bracket Order

Entry: marketable limit (ask + buffer, tick-rounded, outsideRth=True)

Stop: native IBKR stop at stop_price

▼rejected / no fill by close

SKIPPED

entry_rejected or
entry_expired_unfilled

▼fill + stop confirmed

OPEN

plan.json status=open
exit_deadline_at set

▼fill but no stop

FLATTEN

stop_placement_failed
exit immediately

▼

Monitor (3 checks, every 30s — in this order)

1. Position gone?
stop fired → record + skip rest

→

2. Account < 85%?
kill-switch → flatten

→

3. Market close?
max hold → flatten

Order matters: check position-gone first to avoid flattening a non-existent position

▼one of 3 triggers

CLOSED

Write trade/result.json (P&L + all prediction fields + mode flags)

Update plan.json status=closed

▼

Learner reads this later

result.json is self-contained — learner doesn't need plan.json

Recovery (on daemon restart)

plan.json status = entering

Position + stop exist → promote to open

Position exists, no stop → FLATTEN

No position, no order → skipped.json

No position, order working → keep entering

plan.json status = open

Position + stop exist → resume monitoring

Position exists, no stop → FLATTEN

No position → record as stop_fired

Safety invariant: There is never a live position without a broker-native stop. If stop is missing → flatten immediately. Broker stop protects even when daemon is down.

Signal-Agnostic Architecture

Canonical Signal Contract

TES reads only generic fields:

signal_id, source_type, symbol, event_id issued_at, event_time, direction confidence, expected_move_range thesis_summary, context_ref, session_id

Source-specific data stays behind context_ref

Phase 1: Earnings Adapter

Maps earnings prediction → canonical:

prediction_id → signal_id ticker → symbol quarter_label → event_id predicted_at → issued_at filed_at → event_time source_type → "earnings"

Future: news adapter, macro adapter, etc.

Files

trade/plan.json

Live trade state. Daemon-owned.

signal_id, status, shares, entry/stop prices, ATR, spread, pre-close, deadlines, mode flags, session refs

trade/result.json

Final P&L. Written once on close.

Self-contained: P&L + prediction fields + stop params + mode flags + timestamps

trade/skipped.json

Gate rejection. Written once.

reason code + signal identifiers

Idempotency: if any trade file exists for a signal_id → daemon skips. No duplicate trades. No reprocessing.
Atomic writes: write to .tmp → fsync → rename. Crash-safe.

Dependencies & Config

Runtime Dependencies

IBKR Paper Gateway
Neo4j (ATR data)
exchange_calendars

No Redis. No Claude API. No streaming. File lock for singleton.

Startup: lock file → SIGTERM handler → connect IBKR → create/reuse state file → recover open trades → loop

Currency: paper account is CAD, stocks are USD. Phase 1 uses net_liquidation as-is (no conversion). Acceptable for paper validation.
