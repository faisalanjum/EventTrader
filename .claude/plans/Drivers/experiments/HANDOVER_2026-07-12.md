## 1. What this project is (plain terms)

One shared catalog of **Drivers** ‚Äî reusable named causes that move stocks (`same_store_sales`, `oil_price`,
`aws_outage`). A **DriverUpdate** is one real fact about a driver in one event, stored with a deterministic id
(**event + driver + fact_scope**) that never changes and never moves. The same cause always gets the same name,
so mentions line up into one clean per-cause history across companies and time; verdicts (`EXPLAINED_BY` edges)
grade each cause against the realized stock move. Two prior versions died: v1 (closed word-list ‚Üí rejected 82%
of valid names) and v2 (eager merging ‚Üí ~99% string metrics, ~29% judged precision). This version: **AI judges
meaning, code checks structure, always lean specific; breadth only by emergence.**

## 2. Architecture in depth (current, post-2026-07-11 incorporation)

**Identity model.** A Driver stores exactly ONE lower_snake name (NAME-05/06); duplicates found later join via
reversible `SAME_AS` edges (both nodes survive; nothing deleted). A fact's id = event + driver + **fact_scope** =
slice + period + measurement (+ per-X in the NAME, + rare quote-hash). Facts never move or re-key ‚Äî a wrong
exact-name ATTACH is the ONE physically irreversible error; the whole guard stack exists for it.

**Naming (02, NAME-01..19 + OD-17).** Name = cause only; nothing else ever (NAME-01/15/16). As specific as the
evidence allows (NAME-04); familiar macro names win only for undifferentiated mentions ‚Äî a stated sibling
instrument beats the umbrella (NAME-07 precedence, landed 2026-07-11: SOFR ‚Üí `sofr_rate`, never `fed_rate`).
Own measured company parts ‚Üí slice, never the name (NAME-10); external actors/objects ‚Üí name (NAME-11; NAME-16 #4
was REWRITTEN 2026-07-11 to the external-actor principle: `walmart_price_cuts`, `aws_outage`, `tiktok_ban` legal;
banned only for the reporting company itself and incidental co-mentions ‚Äî 95 #40). Singular-by-default with an
objective standard-term test (NAME-06 note). **OD-17**: portion qualifiers (`current_rpo`, `funded_backlog`) stay
in the NAME and are never SAME_AS the bare form; (a) omitted slice only if population = consolidated reporting
entity ‚Äî systemwide/GMV/TPV are their own whole-phrase Drivers; (b) residual buckets (`segment:other`) are legal
company-specific slice values (composition drifts; excluded from future cross-company folding); (c) accounting
constructs (eliminations) excluded as names AND slices ‚Äî the mover routes to the affected reported metric.

**Slices & fact_scope (03, FS-01..26).** Six kinds (segment/product/geography/customer/channel/entity_ownership)
+ `unknown` fallback; operate/sell test; frozen XBRL axis‚Üíkind table (~55 axes), 3-way sentinel (known / non-slice
/ PROVISIONAL hex-encoded unknown-axis), per-company PIT menus (union of prior filings, cut ‚â§ event time), FS-15

values immutable (FS-17); code never near-snaps (FS-16). Measurement = code-normalized token set inside
fact_scope (OD-9: producer copies spans, code normalizes, maximal contiguous span = one token; "contiguity"
defined 2026-07-11 = separated only by punctuation/whitespace; no write-time synonym merging ever).
**FS-26 CONTINUES_AS (new 2026-07-11)**: company-scoped, directional, dated continuity declaration created ONLY
