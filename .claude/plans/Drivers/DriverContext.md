## 1. What this project is (plain English)

Build **ONE shared catalog of "drivers"** — named, reusable causes of stock-price moves (`same_store_sales`, `oil_price`, `food_safety_incident`) — used by all producers (earnings-learner, news producer, predictor-as-consumer).

How: blind AI readers coin candidate names from real company source text (filings, transcripts, fiscal.ai KPIs — **never news**; news drivers accrete LIVE in production). Deterministic code groups them into per-driver evidence records. A reconcile stage (AI dedup ‖ AI gate → AI Refute-skeptic → deterministic assembly → deterministic validator) produces an industry catalog. Industry catalogs fold bottom-up: Company → Industry → Sector → Global, re-using the same reconcile at every level.

The purpose triad (DriverExperiment.md "Primary priorities" — everything serves this):
1. **PURPOSE**: traceable + tradeable inventory — each driver specific enough to trace over time and trigger buy/sell workflows.
2. **MINIMAL but SAME-NAME-WHEN-SAME-MEANING** (+ inverse: ONE NAME = ONE MEANING; mixed-meaning collisions get D5 review, never silent union). Duplicates link via reversible `SAME_AS`, never destructive merge.
3. **LIVE REUSE ORDER (G1, propose-first)**: producer coins its own name BLIND first → then sees retrieved related catalog names (company → industry → sector → semantic) → reuses only on EXACT meaning → new names pass the G2 gate.

## 2. Why the design is what it is — the two deaths + locked invariants

- **v1 died**: closed vocabulary + Python `canonicalize()` rejected **82% of even-correct names** (measured). Lesson: open vocabulary; names come from the source; never bring the closed vocab back. The full v1 design survives only in `archive/` (§10).
- **v2 died**: aggressive merging collapsed different demand themes into a generic `revenue_demand` → false cross-company read-throughs. Lesson: **the worst failure is a wrong merge** (unrecoverable; becomes a bad trade). Under-merge is recoverable (repair pass re-suggests forever).

Locked invariants (do not reopen):
- **Fail-close**: err specific; missing/refuted verdict NEVER fuses; UNCLEAR parks; default keep-separate.
- **3-check for any fusion** (SAME_AS / reuse / rewrite / D5 SAME union): same object · same scope · same mechanism — any false/unclear ⇒ keep separate. Both reviewers judge from EVIDENCE (quote/source/date), never the bare name string.
- **Judgment → LLM; structure → code, never the reverse.** The only deterministic decisions are structural (union, count, set-compare, hard-fail). One conscious supersession: catalog ASSEMBLY is deterministic (`assemble_catalog.py`, §11.19) — it applies skeptic-approved maps; it decides no meaning.
- **Refute merge-skeptic** on every fusion arm (dedup SAME_AS, meaning-changing rewrite, D5 same-name SAME union). High-blast fusions (≥8 companies, or global fold with ≥2 children) get a SECOND, perspective-forced (object/scope/mechanism each quote-backed), AND-voted skeptic with a disjoint evidence view. Never majority voting.
- **No route/kind/fundamental-vs-news buckets** — producer concerns, not catalog properties. G2 = reuse/admit/rewrite/skip. Rewrite = wording only. Skip by CLASS not count (a reusable event class like `government_shutdown` is admitted even if seen once; only single-instance-bound names are skipped).
- **No deterministic name-matching code to block bad names** (tickers/brands are common words — `WING`, `SHAK`, "shake" — would false-positive). Hard-bad names = the gate's job (R7); reinforce the PROMPT, never add matching code.
- **Rule changes must be general principles, never sector-specific examples** (examples overfit = how v1 died).
- **Brand/segment metric ≠ company-wide metric** (`taco_bell_same_store_sales` vs `same_store_sales`): separate drivers, never SAME_AS.
- Cross-company peer grouping = retrieval at read time, not coined generic names.

        40k budget; byte-exact conservation; chunks/ + chunks_manifest.json; --verify re-proves from disk)

        readers Write menus/<cid>.json; §8.7a no-chunk-lost hard-fail; A2 resume via resume_menus.py)
  → build_seed.py                                         (deterministic Converge: group by norm(name),
        5-tuple evidence dedup, first-non-null xbrl wins; --expect count cross-check; writes seed.json)
  → reconcile.js                                          (Guard+billing → slice_seed.py ALWAYS → per batch:
        dedup ‖ gate (Opus, evidence-based) → Refute skeptic → code-computed blast counts → ≥8-company

## 8. VERIFIED NUANCES WRITTEN NOWHERE ELSE (the deep-read's unique findings)

Code-level (all verified at file:line during this session):
1. **`reconcile.js` does NOT throw on VALIDATION FAILED** — the Validate agent has no schema; JS throws only if the agent dies.
