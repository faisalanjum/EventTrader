## 0. Plain-English summary (crux)

We build the catalog from the bottom up, like merging folders:

```
                         GLOBAL catalog                 ‚Üê merge all SECTOR catalogs
                   = reconcile( all sector catalogs )
                    ‚ñ≤            ‚ñ≤            ‚ñ≤
              SECTOR catalog (one per sector)           ‚Üê merge that sector's INDUSTRY catalogs
              = reconcile( its industry catalogs )
                 ‚ñ≤        ‚ñ≤        ‚ñ≤
           INDUSTRY catalog (one per industry)          ‚Üê TODAY: blind bots + reconcile
        = menu_build (blind bot per company) + reconcile
              ‚ñ≤     ‚ñ≤     ‚ñ≤
            co.   co.   co.                             ‚Üê leaf: each company named blind
```

- **Generation happens ONCE, at the leaf** (a blind bot per chunk file coins candidate names ‚Äî one bot per company when small, several for a big filer) ‚Äî *one narrow exception: the D5 homonym split (¬ß3c) re-specifies a name above the leaf, drawn only from the colliding occurrences' own evidence ‚Äî and a flag-triggered D5 also runs AT the leaf (¬ß2a.5).*
- **Consolidation (`reconcile`) happens at EVERY level up.** The same dedup ‚Üí gate ‚Üí Refute ‚Üí validate runs each level; only the *units* change (companies ‚Üí industries ‚Üí sectors).
- **The merge rule is bounded context:** keep every AI step small enough to judge carefully. Tiers are just batches; split smaller whenever a step is too big.
- **Worst failure = a wrong merge** (it becomes a false cross-company read-through ‚Üí a bad trade), so every merge is fail-close (default keep-separate) and every merge is **approved by AI + Refute, then assembled AND structurally checked by code** (code checks approval + losslessness, never semantic truth ‚Äî ¬ß11.19).

## 2. The pipeline, level by level (exact procedure)

### 2a. LEAF ‚Äî Company ‚Üí Industry catalog (today, + hardening)

1. **Resolve** ‚Äî `resolve_driver_scope.py --industry X` ‚Üí tickers + a `run_id`.
2. **Fetch** ‚Äî `fetch_company_sources.py --run-dir` ‚Üí all non-news sources per company, emitting **FULL structured content** (caps removed, D7/¬ß8; sha256-pinned in `sources/`). Then a **Chunk** phase (`chunk_company_sources.py`) writes bounded `chunks/` ‚Äî splitting big companies + giant events at natural boundaries.
3. **Menus** ‚Äî one **blind** bot **per chunk file** ‚Üí candidate `driver_name` + `evidence_refs` (each `{company, source_type, source_id, date, quote}`), tagged with the company ticker. *(A company's chunks/parts re-union at Converge.)*
4. **Converge (deterministic code ‚Äî `build_seed.py`)** ‚Äî group by **lower-cased** `driver_name` ‚Üí one self-canonical seed record per name; union `evidence_refs` (dedup by the 5-tuple); `companies = distinct(evidence.company)`. **Write `seed.json` with code** (not via an LLM).
5. **Reconcile** ‚Äî dedup (same-meaning SAME_AS) ‚Äñ G2 gate ‚Üí Refute ‚Üí **deterministic code assembles `catalog.json`** (`assemble_catalog.py`; a thin agent only RUNS the CLI ‚Äî ¬ß11.19/¬ß10(1)) **+ `approved.json`** (the Refute-approved links); sets `canonical_name` + `same_as_variants`. *(High-blast fusions get a 2nd AND-voted Refute ‚Äî ¬ß11.18. **Leaf D5, flag-triggered:** a same-name union whose evidence dedup/gate FLAG as MIXED-meaning routes through the ¬ß3c review ‚Äî SAME ‚Üí Refute-confirmed keep ¬∑ DIFFERENT ‚Üí split per assignment map ¬∑ UNCLEAR ‚Üí park to `unresolved_same_name[]`; wired in Phase 1 with the D5 step. Post-write, the on-disk `catalog.json` must hash-match the code-assembled JSON ‚Äî ¬ß11.19.)*
6. **Validate** ‚Äî `validate_catalog.py seed catalog approved.json` ‚Üí HARD-FAIL on any structural break, **including the D1 fusion-approval check**. Leaf accounting includes `unresolved_same_name[]` + D5 split-map provenance (¬ß11.3).

‚Üí Output: an **Industry catalog** (records + `same_as_variants`, `approved.json`, `validation.txt`), pinned by git commit + source sha256.

### 2b. HIGHER LEVELS ‚Äî Industry‚ÜíSector and Sector‚ÜíGlobal (the fold; NEW)

Same `reconcile`, but the **input is built by a new deterministic combine step**, and a **same-name review** sits in front of it.

1. **Combine part A ‚Äî `fold_catalogs` collapse + queue (deterministic, zero AI):** take N child catalogs ‚Üí for each child resolve every SAME_AS cluster to its self-canonical **representative**, **union all variant `evidence_refs` onto the rep** (dedup by the 5-tuple), recompute `companies`, **carry the cluster's variant names into `same_as_variants`** (D4), and **drop `skips[]`/`unresolved_rewrites[]`/`unresolved_same_name[]`** (fail-close; the trail stays in the committed child catalog). Then group reps **across children by lower-cased `driver_name`**: a name in only one child passes through; an **identical name in ‚â•2 children is NOT merged ‚Äî it is pushed to the SAME-NAME REVIEW queue**, each child's evidence labelled (D5). **No parent seed is written yet.**
2. **Same-name review (AI) ‚Äî D5:** for each queued collision, judge from the evidence ‚Üí **SAME** (a union proposal ‚Äî an independent **Refute** must then fail to break it, else it falls to UNCLEAR; fail-close) ¬∑ **DIFFERENT** (propose more-specific source-grounded names + the evidence assignment map) ¬∑ **UNCLEAR** (mark park). Writes final post-Refute verdicts to `same_name_review.json` = reviews + split-map + assignments (¬ß11.6).
3. **Combine part B ‚Äî assemble the parent `seed.json` (deterministic, code):** apply the review ‚Äî SAME ‚Üí one Refute-confirmed unioned record (5-tuple-union evidence, recompute `companies`); DIFFERENT ‚Üí split names as separate records using the assignment map (they must still clear G2/Refute in step 5); UNCLEAR ‚Üí `unresolved_same_name[]`. Merge `optional_links` per key (**identical ‚Üí keep; conflict ‚Üí null + note**, never silently pick ‚Äî ¬ß11.7). Flatten chains to a STAR; sort catalog by name + evidence by the 5-tuple. Write the seed in **leaf shape + `same_as_variants`** (¬ß11.5).
4. **Fold validator (deterministic) ‚Äî D8:** confirm nothing was dropped or lost (a‚Äìd; accounting set ¬ß11.2). HARD-FAIL otherwise.
5. **Reconcile (same pipeline) ‚Äî D6:** dedup proposes new cross-child SAME_AS for **different** names that mean the same ‚Üí G2 ‚Üí Refute ‚Üí deterministic code assembly (`assemble_catalog.py`) writes `catalog.json` + `approved.json` (+ `same_as_variants` updated). *(DIFFERENT-split names are gate/Refute-validated here before entering the catalog.)*
6. **Validate** ‚Äî `validate_catalog.py` incl. D1 fusion check + the `same_as_variants` check + the D5 provenance / `unresolved_same_name` accounting (¬ß11.3).

‚Üí Output: a **Sector catalog** (then a **Global catalog** by repeating with sectors as the children).

**Why this is safe:** the combine is pure code and *never invents a merge of its own* ‚Äî it only collapses links Refute already approved and routes identical-name collisions to review. New merges only happen through dedup‚ÜíRefute (fail-close) or the Refute-confirmed same-name decision. An **unapproved or lossy** merge cannot pass `validate` (D1 + D8); a semantically-wrong-but-approved merge is **guarded ‚Äî not provably blocked ‚Äî** by AI review + Refute + fail-close.

## 3. New components (exact specs)

### 3a. `approved.json` + fusion-approval check (D1) ‚Äî *do this FIRST*
- **`reconcile.js`** writes `<run_dir>/approved.json` = `{ "same_as": [{variant, canonical}], "rewrites": [{from, to}] }` (the `survivingLinks` / `appliedRewrites` it already computes).
- **`validate_catalog.py`** takes an optional 3rd arg `approved.json`. New check: for every catalog record with `norm(canonical_name) != norm(driver_name)`, the pair must appear in `approved.same_as` (variant‚Üícanonical) OR `approved.rewrites` (from‚Üíto) ‚Äî else HARD-FAIL. Pure set membership, zero judgment.
- Runs at **every** level (leaf + every fold) automatically, since every level runs reconcile + validate.

### 3b. `fold_catalogs` (NEW deterministic combine) ‚Äî see ¬ß2b step 1
- Pure **JS** (¬ß11.13), no AI, no new merge judgment. Mirrors `menu_build`'s Converge, one level up. Numbered algorithm in ¬ß11.7.
- Outputs: the parent `seed.json` + the same-name-review queue + (later) `unresolved_same_name[]`.
- Deterministic: sorted everywhere; reproducible given the same child set.

### 3c. Same-name review (D5)
- Input: one normalized `driver_name` present in ‚â•2 children ‚Äî **or, at the LEAF, a same-name union FLAGGED as mixed-meaning by dedup/gate (flag-triggered only; never a blanket review)** ‚Äî with a **representative view** of each occurrence's evidence (¬ß11.17/¬ß12.8; full evidence stays stored + validated). **Occurrence key:** `child_run_id` at a fold, `company` at the leaf.
- AI verdict: **SAME ‚Üí an independent Refute must confirm the union** (it tries to BREAK it; can't-refute ‚Üí union, refuted-or-missing ‚Üí UNCLEAR; a **high-blast** union gets a 2nd AND-voted Refute ‚Äî ¬ß11.18; the merge direction is the dangerous one) ¬∑ DIFFERENT ‚Üí split (more-specific names **drawn only from the existing evidence**, then G2/Refute) **+ an evidence-assignment map (¬ß11.6): which child/refs ‚Üí which new name** (default per `child_run_id`; code applies it to ALL refs; D8 verifies every `from` ref is assigned exactly once) ¬∑ UNCLEAR ‚Üí park.
- The **only** name-generation allowed above the leaf. Rare (the ontology already pushes specific names, so true homonyms are uncommon).

### 3d. Fold validator (D8)
- Extend `validate_catalog.py` with a **fold mode** (CLI in ¬ß11.9). Checks D8 (a‚Äìd) above ‚Äî accounting unit = **names**, exact set in ¬ß11.2. **Required validator changes so D5 can't crash it (¬ß11.3):** (1) PROVENANCE must accept D5-split target names (read the split-map, like `approved.json`) ‚Äî they are legitimately non-seed; (2) COMPLETE must count `unresolved_same_name[]`. HARD-FAIL on any break. Read-only, additive, fail-loud.

### 3e. `same_as_variants` field
- On every self-canonical **CATALOG (output) record**: `same_as_variants: [names]` = all variant names linked to it, accumulated across levels.
- Written by: deterministic code catalog assembly (`assemble_catalog.py`) in reconcile (from `approved.json`); `fold_catalogs` (carry-forward + merge); higher reconcile (new links). Kept honest by the fold validator (d).
- **Provenance** (which child/industry each record came from) is **derivable** from `evidence_refs[].company` ‚Üí its industry/sector ‚Äî no separate field needed unless we choose to materialize it.

## 5. Invariants & fail-close guarantees (must always hold)

- **Fail-close:** a missing or refuted verdict NEVER fuses; default keep-separate; UNCLEAR parks. Looping/folding can only ADD skeptic-survived structure, never relax it.
- **Judgment ‚Üí LLM, structure ‚Üí deterministic code.** `fold_catalogs`, both validators, and Converge decide nothing about meaning; they only union, count, set-compare, and HARD-FAIL.
- **An unapproved or lossy merge cannot ship.** D1 (every fusion's `canonical_name` must be in the Refute-approved set) + D8 (no record/evidence lost in the combine) make THAT a CODE invariant. *Code proves structure (approved + lossless), not meaning* ‚Äî the residual risk of a semantically-wrong-but-**approved** merge is **minimized, not eliminated**, by independent AI review + Refute + fail-close defaults ‚Äî **deepened on the highest-blast fusions by a second AND-voted Refute + an adversarial evidence view (¬ß11.17/¬ß11.18)**. And because the catalog is **assembled by deterministic code** (`assemble_catalog.py` ‚Äî ¬ß11.19/¬ß10(1)), the writer **cannot fabricate** an unapproved `canonical_name` ‚Äî the fusion hole is structurally closed; D1 is belt-and-suspenders.
- **SAME_AS is reversible; never a destructive merge.** *Reversible* = the link can always be undone and the originals reconstructed. At the **leaf**, both records survive in place. At a **fold**, the variant collapses to a NAME in `same_as_variants` (its evidence unioned onto the rep) and its full record stays recoverable from the committed child catalog. Trading reads the cluster via `same_as_variants`.
- **No new drivers above the leaf**, except the narrow source-grounded homonym split (D5).
- **Reproducible:** pinned git commit + source sha256 + a snapshot of the sector/industry taxonomy used; deterministic (sorted) fold. *(Note: the generated candidate NAMES are LLM output ‚Äî the pipeline is reproducible; the exact vocabulary a re-run coins is not.)*
- **Cost:** all AI runs **in-session (subscription)**; embeddings (if ever) are OpenAI + suggest-only. NEVER `claude -p` / `claude_agent_sdk`.
- **Read-only Neo4j** during calibration.

## 8. Chunking (D7) ‚Äî LOCKED (revised after measuring clipping)

**Why this changed:** sampling the current graph showed today's caps clip ~**62%** of report/transcript events (~52% reports, ~**100%** transcripts). Dropping that much text is a real recall loss, so we **stop dropping text**: `fetch` emits full content, and chunking bounds each bot's load by **splitting**, never by clipping. ("Measure first" ‚Üí measured ‚Üí caps removed.)

**Goal:** never drop source text; keep every blind bot's reading load bounded. Touches ONLY the leaf.

1. **Originals stay the truth.** `runs/<run_id>/sources/<TICKER>.json` = the raw fetched artifact, sha256-pinned, untouched.
2. **`fetch` emits FULL structured content (caps REMOVED).** Delete the **full set** of caps ‚Äî `CAP_EVENT_REPORT`, `CAP_EVENT_TRANS`, `CAP_SECTION`, `CAP_EX991`, `CAP_PREPARED`, **`CAP_QA_EACH`** (the per-Q&A 900-char slice ‚Äî the main cause of the ~100% transcript clipping), `QA_MAX=8`, plus the `substring()`/`[..]` slices in `TRANS_Q` and the final `[:CAP_EVENT_*]` slices in `build_report_content`/`build_trans_content` ‚Äî and emit each event as the natural sub-units `fetch` already builds ‚Äî full `prepared`, ALL `qa_exchanges[]`, full `sections[]`, full `ex991` ‚Äî with `source_id`/`source_type`/`date` intact. **Sort Q&A numerically by `toInteger(qa.sequence)`** ‚Äî `qa.sequence` is stored as a STRING (`"0"`,`"1"`,`"10"`), so the current `ORDER BY qa.sequence` sorts it alphabetically and scrambles Q&A order; this is a required `fetch` fix. (Removing the caps also makes `fetch` simpler ‚Äî it stops capping + joining.)
3. **New helper `workflows/chunk_company_sources.py`** (derived layer): `sources/<TICKER>.json` ‚Üí `runs/<run_id>/chunks/<TICKER>__chunk_NNN.json` + `chunks_manifest.json`. Deterministic, **fewest chunks**. Neo4j confirms stable boundaries (`HAS_SECTION` / `HAS_EXHIBIT`; `PreparedRemark` + `QAExchange.sequence`) **but some single blocks are huge** (EX-99.1 ‚âà1.96M chars ¬∑ RiskFactors ‚âà544k ¬∑ prepared ‚âà183k ¬∑ one Q&A ‚âà54k) ‚Äî so natural boundaries ALONE are not enough. Use a **layered fallback ladder** that ALWAYS finds a split point, so **no text is ever dropped**:
   - **(a) company-level** ‚Äî group whole small events into a chunk until the per-bot budget is reached.
   - **(b) natural unit** ‚Äî split a too-large event into units: report ‚Üí section / exhibit; transcript ‚Üí prepared remarks / each Q&A exchange; KPI ‚Üí never split.
   - **(c) paragraph** ‚Äî if one natural unit is still > budget, split it by paragraph.
   - **(d) sentence** ‚Äî if one paragraph is still > budget, split by sentence.
   - **(e) char range** ‚Äî if one sentence is still > budget, split by char range (last resort; astronomically rare).
   - Order is preserved at every level; KPIs in `chunk_001` only; the chunker **slices bytes, never mutates them**. Because (e) always succeeds, **no text is ever dropped** ‚Äî proven by the byte-exact conservation check (¬ß8.7c).
4. **`menu_build.js`:** add a **Chunk phase** after Fetch (runs the helper); **Menus = one blind bot per chunk file** ‚Äî each bot sees either a group of whole small events OR one part of one large event, and nothing else (no other company, no catalog). The **intermediate** menu result carries its `chunk_id` + `ticker` (the `chunk_id` is matched against `chunks_manifest.json` for the processed-check in step 7 ‚Äî it is **NOT** added to the final `evidence_ref`).
5. **Combine = Converge (unchanged).** A company's chunks/parts auto-union by exact `driver_name`; candidates coined in different parts re-union here. No new combine code.
6. **Part-tracking lives in the manifest, NOT the record.** `chunks_manifest.json` records `ticker, source_id, part_index, part_count, split_level (whole|natural|paragraph|sentence|char), char-range, sha256` ‚Äî proving every unit/part was produced and processed. The final `evidence_ref` keeps its locked 5-tuple (`company, source_type, source_id, date, quote`) ‚Äî **no new `evidence_ref` fields, no part-tracking in the record** (part data lives in `chunks_manifest.json`). *(The record does gain `same_as_variants` ‚Äî that is the read-through field, unrelated to chunking; see ¬ß9.)*
7. **Reliability HARD-FAILS (deterministic):** (a) every `chunk_id` in `chunks_manifest.json` appears in a returned menu result (closes the `.filter(Boolean)` gap ‚Äî no chunk silently dropped); (b) for any split event, **all parts `1..part_count` are present**; (c) **byte-exact conservation** ‚Äî for every event, the ordered concatenation of its parts **equals the original event text** (compared by hash; **NO whitespace normalization** ‚Äî the chunker *slices*, never mutates bytes); HARD-FAIL otherwise. This is the real no-text-loss proof (it replaces the old `clipped` flag ‚Äî now removed, since a never-firing signal is dead weight once byte-exact equality is enforced).
8. **No semantic decisions in chunking** ‚Äî it only splits text and preserves evidence. G2/Refute still own naming validity + SAME_AS.
9. **Tradeoff accepted:** a split event = a bot sees only part of it (minor cross-part context loss), mitigated by natural boundaries + Converge re-union + a **generous per-bot budget** so splitting fires only for giant 10-Ks. Far better than dropping 62% of the text.

**Cost:** full content = more chunks = more bots (in-session, parallel, bounded). Keep the per-bot budget **high** to minimize chunk count.

**Higher-level batching = the SAME rule (D2/D8):** if any industry/sector/global combine is too big ‚Üí split child catalogs into batches ‚Üí reconcile each ‚Üí combine ‚Üí reconcile again. **Atomic unit = one *event-part* at the leaf, one *Driver record* above.** Every combine validator-checked (D8).

**Recall floor (fold in here):** warn/flag if a company with non-trivial source text yields 0 candidates.

**Build timing:** **Phase 0.5** (¬ß6) ‚Äî foundational, right after the Phase-0 leaf hardening (the whole catalog quality depends on the leaf seeing the full text).

**11.17 Reconcile evidence-view cap [OWNER].** A single record can accumulate huge evidence at scale (e.g. `oil_price` across hundreds of companies). **11.18 Blast-radius-gated second Refute (AND-only, perspective-forced) [OWNER].** A leaf wrong-merge hits 2‚Äì3 companies; a global one hits hundreds + many trades ‚Äî v2 died on exactly a high-fan-out over-merge. The fusion survives only if the **first** Refute survives **AND** the perspective-forced 2nd Refute passes all three lenses (**AND-vote** ‚Äî any FALSE or missing ‚Üí keep separate / UNCLEAR). The 2nd Refute is shown a **disjoint evidence view ‚Äî ¬ß12.8 view-2: for each side, the next refs in the same deterministic sequence, never re-showing view-1 refs** ‚Äî so wherever enough refs exist it sees quotes the first reviewer did not, closing the sampling-blindness the ¬ß11.17 view names. One Refute everywhere else, unchanged. The AND-filter is deterministic JS; the per-fusion company counts are COMPUTED BY CODE everywhere ‚Äî at the LEAF by a python one-liner over the batch file/seed (agent = relay only), at FOLDS by the deterministic part-a `collision_meta` ‚Äî and ENFORCED by the validator backstop: `validate_catalog.py` recomputes counts from the seed and HARD-FAILS any >=8-company approved fusion lacking a surviving second-skeptic verdict in `approved.json.high_blast_refute2` (12th pass rev2). **Never majority voting** (majority would relax fail-close in the over-merge direction). **Same agent, same cost** (one extra call on the rare gated fusions).

**11.19 Deterministic catalog assembly (the writer = `assemble_catalog.py` ‚Äî see ¬ß10(1); "JS" wording below is historical).** The reconcile/fold writer is now pure structure ‚Äî apply the skeptic-approved maps (SAME_AS / rewrites / skips / D5 assignments) by fixed precedence and copy evidence verbatim. **11.20 Full-tree (`--list`) mode ‚Äî acceptance criteria (frozen NOW so Phase 2 is never re-litigated).** **Key code fact:** `resolve_driver_scope.py --list` returns only TWO **flat** lists `{sectors, industries}` with **no parent-child links** (verified, lines 40-43; live shape = strict **11 sectors ‚Üí 115 industries ‚Üí 796 tickers**, every industry‚Üí1 sector, every ticker‚Üí1 industry, zero nulls). ## 12. Build-time contracts (exact parseable shapes ‚Äî removes builder guessing)

**12.1 ‚Äî One shared `norm()`.** `norm(s) = s.strip().lower()` (ASCII) ‚Äî already `reconcile.js:73` + `validate_catalog.py:27`. The **same** key is used by **menu_build Converge** (E1 = change `.trim()` ‚Üí `.trim().toLowerCase()`), the **deterministic writer** (`assemble_catalog.py` ‚Äî ¬ß11.19), **`fold_catalogs` grouping**, and the **D1 fusion check** (¬ß3a). **12.2 ‚Äî `event_text` + byte-slice chunking** (serves ¬ß8 / ¬ß8.7c / D7). The chunker forms, per `source_id` event, **`event_text`** = the event's sub-units joined in the existing `build_report_content` / `build_trans_content` order, **uncapped** (report: `FORM` ‚Üí `ITEMS` ‚Üí optional `DESC` ‚Üí earnings `PRESS RELEASE (EX-99.1)` ‚Üí each `[section]`, blocks joined `\n\n`; transcript: `PREPARED REMARKS` ‚Üí `Q&A:` with exchanges joined `\n---\n`, blocks joined `\n\n`). The ¬ß8 ladder **slices `event_text` at UTF-8 byte offsets that are valid character boundaries** ‚Äî natural-unit rung = at sub-unit boundaries, finer rungs subdivide a too-big slice ‚Äî so **every byte (including separators) lands in exactly one part, in order**. ¬ß8.7c check = `concat(parts in order) == event_text`, byte-for-byte. The chunker **slices the joined string; it never re-serializes sub-units.**

**12.3 ‚Äî Perspective-forced 2nd Refute schema** (serves ¬ß11.18). The 1st Refute is unchanged (`survives:bool`). The high-blast 2nd Refute returns `{ object:{pass:bool, quote}, scope:{pass:bool, quote}, mechanism:{pass:bool, quote}, survives:bool }` with `survives == object.pass AND scope.pass AND mechanism.pass`; each lens applies the existing `EXACT_MEANING_RULE` (reconcile.js:15-21). Its evidence view is deterministic: **the NEXT up-to-20 refs in the ¬ß12.8 sequence** (continue the same sequence, skipping every ref shown to the 1st Refute); if fewer remain, the view is simply shorter ‚Äî never re-show view-1 refs, never pad. A missing field or `survives ‚âÝ true` ‚Üí keep-separate (fail-close).

**12.4 ‚Äî Same-name review parse contract** (design is in ¬ß11.6/¬ß3c ‚Äî NOT restated here). Agent output = `{ reviews:[{collision_name, verdict:"SAME"|"DIFFERENT"|"UNCLEAR", new_names?:[‚Ä¶], why}], split_map:[{from, to:[names], assignments:[{child_run_id, to, ref_idx?}]}] }`. Evidence view = the ¬ß12.8 draw (‚â§ 20 refs per colliding occurrence), each ref displayed with its deterministic `idx` (`r1, r2, ‚Ä¶` ‚Äî ¬ß11.6 assignment interface; the AI assigns indices, code maps them back to full refs). At the leaf, `child_run_id` is replaced by `company` (¬ß11.6). **Leaf trigger flag:** dedup/gate may emit `mixed_same_name_flags:[{driver_name, source:"dedup"|"gate", why}]`; ONLY normalized names in this side-list route to leaf D5 review, and absence from the side-list means the normal same-name Converge union stands.

**12.5 ‚Äî chunk-file + chunks_manifest schemas** (serves ¬ß8). chunk-file (`chunks/<TICKER>__chunk_NNN.json`, read by the blind bot) = `{ ticker, chunk_id, fiscal_kpis?:[‚Ä¶] (chunk_001 only), events:[{source_id, source_type, date, part_index, part_count, content}] }` (`content` = a ¬ß12.2 slice; a whole event ‚Üí `part_index = part_count = 1`). `chunks_manifest.json` (read by the ¬ß8.7 HARD-FAILs), one row per emitted part = `{ ticker, chunk_id, source_id, part_index, part_count, split_level:"whole"|"natural"|"paragraph"|"sentence"|"char", char_start, char_end, sha256 }`; `sha256` = SHA256 of the part's UTF-8 `content` bytes.

**12.8 ‚Äî The ¬ß11.17 evidence-draw algorithm (deterministic; used by reconcile dedup/gate/Refute prompts, the same-name review, and the ¬ß12.3 2nd-Refute view).**
1. **Sides** = the input units of the decision under review: for a SAME_AS / rewrite pair ‚Üí the two records; for a same-name review ‚Üí each occurrence (`child_run_id` at a fold, `company` at the leaf); for a single-record gate view ‚Üí one side (degenerate).
2. **Canonical ref order** (within everything): sort `evidence_refs` by the 5-tuple `(company, source_type, source_id, date, quote)`, lexicographic.
3. **Side order:** ascending total ref count ‚Äî **the smaller side of a proposed merge always contributes first** (tie ‚Üí lexicographic side key).
4. **Within a side:** round-robin over its companies in ascending per-company ref count (tie ‚Üí lexicographic company); within a company, ref priority = one ref per distinct `source_type` first (source_types sorted; pick each type's earliest-date ref, empty date sorts first), then earliest-date remaining, then latest-date remaining, then the rest in canonical order.
5. **Draw ‚Äî the cap is PER SIDE** (the locked `RECONCILE_EVIDENCE_PER_RECORD` = 20, owner setting #3 "20 refs/record"): each side independently contributes up to 20 refs via steps 2+4; sides are PRESENTED in step-3 order (smallest side first), so the smaller merge side is always read first and can never be drowned by a big side. (A single-record view = steps 2+4 alone.)
6. **View 2 (¬ß12.3 / ¬ß11.18):** continue the SAME per-side sequences ‚Äî for each side, the next up-to-20 refs not shown in view 1; if fewer remain, the view is shorter (never re-show, never pad).
The sequence is a pure function of the stored refs ‚Äî two builders produce byte-identical draws. ("Minority force-include" is implemented structurally by steps 3‚Äì5: the smaller merge side and the least-represented companies always contribute before any large side adds a 2nd ref.)

## 13. Consumption & Lifecycle Requirements (owner-approved 2026-06-10 ‚Äî the "after the build" contract)

> These do NOT change the batch-build algorithm. They constrain what the build must keep clean (`same_as_variants`, evidence, canonical clusters) and define how the catalog is used, repaired, ranked, and refreshed. Algorithm choices below were owner-reviewed with the integrity-first rule: judgment ‚Üí LLM, mechanics ‚Üí code, embeddings/counts suggest-only, fail-close everywhere.

**13.1 Consumption Contract (producer + learner).**
1. The producer ALWAYS coins its own `driver_name` + exact supporting quote FIRST, blind (G1 ‚Äî unchanged).
2. Only then is it shown related EXISTING drivers, retrieved in four lanes ‚Äî company ‚Üí industry ‚Üí sector ‚Üí global ‚Äî and ranked by SEMANTIC match. **Retrieval embeds `proposed_driver_name + short evidence quote + scope (company/industry/sector)` ‚Äî never the name alone** (names alone agree only ~72%; meaning lives in the quote). Retrieval is **PIT-filtered** (`visible_from` ‚â§ event date).
3. Each candidate displays: canonical name ¬∑ its `same_as_variants` ¬∑ usage counts per lane (company/industry/sector/global) ¬∑ best evidence quote ¬∑ why-shown. **Counts may break ties WITHIN a lane; they never outrank semantic match and never decide reuse.** Embeddings suggest, never decide.
4. The producer's pick (reuse exact-meaning OR keep its new candidate) is still only a PROPOSAL ‚Äî G2 (+ Refute for any fusion) remains the admission authority. New-name admission runs **sync** per name (cheap once the catalog matures; avoids provisional-history renames).
5. **Learner/trading contract:** before using any history, resolve `driver_name` ‚Üí its canonical cluster (follow `canonical_name` + union all `same_as_variants` histories in ONE hop).
6. **Fitness measurement (pre-registered):** reuse-rate + inter-producer agreement WITH the catalog must beat the measured no-catalog baseline (72% name agreement / 0.634 registry / 0.535 blind) on held-out events, graded once.

**13.2 Duplicate-repair pass ‚Äî a REQUIRED pipeline step (no longer deferred).** After every level's reconcile+validate: `build catalog ‚Üí repair pass ‚Üí validate`. Algorithm: **(a) suggest** candidate pairs by the UNION of token-overlap (free, deterministic) and embedding nearest-neighbours (suggest-only; the one metered OpenAI piece ‚Äî pennies per catalog); **(b) judge** each pair with the SAME Refute discipline (3-check, default keep-separate; ‚â•8-company pairs get the ¬ß11.18 second skeptic); **(c) apply** approved pairs by APPENDING them to `decisions.json`/`approved.json` and RE-RUNNING the existing `assemble_catalog.py` + validator ‚Äî no new writer, the proven deterministic path. Constraints: only ADDS links between kept self-canonical records; never touches skips/parks; never reopens existing links; star-flatten consistency comes free from the assembler. This closes the accepted cross-batch SAME_AS residual at every level. Parks remain terminal (out of repair scope).

**13.3 Recurrence tier ‚Äî display-layer minimality (storage untouched).** `recurrence_tier ‚àà {common | occasional | rare}` is computed at **retrieval time** (a VIEW ‚Äî never stored in `catalog.json`, so the locked record shape, validators, and SAME_AS truth are untouched; never deletes anything). v1 thresholds (tunable, cluster-level over canonical + variants): rare = 1 company AND ‚â§2 evidence events ¬∑ common = ‚â•3 companies OR ‚â•8 events ¬∑ else occasional. In production, recomputed from DriverUpdates. Used ONLY to down-rank rare drivers in producer display (13.1.3); rare drivers remain fully admitted, stored, PIT-visible, and tradeable.

**13.4 Incremental refresh ‚Äî first-class, reusing the fold machinery.** Two flows, never conflated: **(a) LIVE per-event flow** = 13.1 (propose ‚Üí retrieve ‚Üí reuse-or-create ‚Üí sync G2/Refute ‚Üí DriverUpdate); **(b) BATCH corpus refresh** (new quarter of filings/transcripts): delta menu_build over ONLY new events (fetcher takes a since-date) ‚Üí delta leaf catalog ‚Üí **merge = `fold(existing_catalog, delta_catalog)`** using the EXISTING fold pipeline (exact-union evidence, same-name review, D8 validation ‚Äî zero new algorithms) ‚Üí repair pass (13.2) ‚Üí propagate by re-folding ONLY the dirty branch (leaf ‚Üí its sector ‚Üí global). Never rebuild the world; seed once, increment forever.

**13.5 Truth-in-spec (applied 12th pass):** "unapproved or lossy merge cannot ship" is the code-proven guarantee; semantically-wrong-but-approved merges are minimized (independent review + Refute + 2nd skeptic + fail-close), not impossible.
