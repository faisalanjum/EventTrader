## §0 Mission, scope, and the one law

- **PIPE-01 — Purpose.** Produce ONE shared Driver **class** catalog: reusable cause-names (e.g. `same_store_sales`, `oil_price`, `revenue_guidance`), each backed by real evidence quotes (one deliberate exception: latent base NAMES in `families.json` — family anchors with no evidence yet, PIPE-25), specific enough to track over time, reused by every producer. `[01_Overview · MF-05]`
- **PIPE-02 — The Track-A deliverable.** A Driver class = **`driver_name` + `fact_type` + family/`SAME_AS` links + evidence**. Nothing else. `[DU-02]`
- **PIPE-03 — CLASS vs FACT (the scoping law).** Everything company/value/time-specific — slice **values**, measurement, DriverPeriod, units, numbers, `driver_state`, the EXPLAINED_BY verdict, XBRL **concept/member links**, write-time PIT menus — lives on the **DriverUpdate fact**, written by producers (Track B/C), never by this build. Track A never writes a DriverUpdate; KPI-only evidence makes a **name only** (DU-02). Any later graph sync that sees an existing fact-bearing Driver (a Driver with any incoming `OF_DRIVER` from `DriverUpdate`) must hard-fail rather than rename, delete, or re-type it; later catalog work may add `SAME_AS`/`BASE_METRIC` links, never re-key stored facts. `[DU-01/02 · FS-21 · XC-01 · 13 §8]`
- **PIPE-04 — The one law.** Over-merge = permanent damage; over-split = a cheap fix. When unsure, keep separate. The LLM judges meaning only (assign / coin / flag) — it never merges two existing identities; code merges only via approved links, deletes only from frozen lists, and validates structure with hard-fails. `[01_Overview · FS-04]`
- **PIPE-05 — Non-goals of this plan.** DriverUpdate creation · slice-value assignment · units/periods/numbers · verdicts · XBRL concept-linking (§6) · live G1 reuse display (`catalog_first.js`) · Neo4j production writes · incremental refresh (§13) · Track C old-guidance archive/retirement · live/backfill guidance production process (part 2).

**PIPE-13 — Gates, exact semantics.**
- **G2 (admission)**: per candidate, from EVIDENCE (never the bare name): **reuse** (exact same cause+scope as an existing catalog name — standalone `gate.js` only; inside reconcile the reuse arm is the dedup lane) · **admit** (valid reusable cause following every naming rule) · **rewrite** (wording-only fix, target must be an EXISTING coined name — never invent) · **skip** (vague / rule-breaking / single-event-bound). Reusable event CLASSES admitted even if seen once (`government_shutdown` OK; `q1_2026_shutdown_effect` skip). No fundamental-vs-news routing.
- **Refute (merge skeptic)**: attacks every proposed fusion (SAME_AS link, meaning-changing rewrite, D5 SAME union). Default `survives=false`; flips ONLY if it cannot refute the **3-check: same object + same scope + same mechanism** (scope lens redefined, PIPE-19). Code filters mechanically: refuted links dropped, refuted rewrites parked, missing verdict = refused.
- **High-blast 2nd skeptic**: any surviving fusion spanning **≥8 companies OR (global fold AND ≥2 children)** gets a second, independent, perspective-forced Refute — separate PASS/FAIL per lens (object·scope·mechanism), each backed by verbatim quotes from BOTH sides, **AND-voted**, shown the disjoint view2. Blast counts computed by CODE; verdicts land in `approved.high_blast_refute2`; the validator recomputes counts from the seed and hard-fails any ≥8-company fusion without a surviving verdict.

## §6 Why XBRL concept-linking is Track B (the investigated answer, kept for the record)
