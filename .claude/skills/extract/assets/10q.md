# 10-Q Extraction Profile

Per-source profile for 10-Q quarterly filings. Loaded by the extraction agent when `ASSET = 10q`.

## Asset Metadata
- sections: full
- label: 10-Q
- neo4j_label: Report

## Data Structure

10-Q is a layered Report source. Use the report inventory query (5F) first to determine which content layers exist for the filing.

| Layer | Node Label | Relationship | Typical Role |
|-------|-----------|--------------|--------------|
| **Canonical MD&A section** | ExtractedSectionContent | `Report-[:HAS_SECTION]->` | Common management narrative section; fetch directly with 5B when needed |
| **Other extracted sections** | ExtractedSectionContent | `Report-[:HAS_SECTION]->` | Additional named sections; discover with 5D and fetch with 5I |
| **Financial statements** | FinancialStatementContent | `Report-[:HAS_FINANCIAL_STATEMENT]->` | Structured statement / footnote payloads |
| **Filing text** | FilingTextContent | `Report-[:HAS_FILING_TEXT]->` | Raw full filing text fallback |
| **Exhibits** | ExhibitContent | `Report-[:HAS_EXHIBIT]->` | Attached exhibit text |

### Report Fields

| Field | Type | Description |
|-------|------|-------------|
| `r.accessionNo` | String | Canonical Report ID for the filing |
| `r.id` | String | Same as `accessionNo` |
| `r.created` | String (ISO) | Public filing timestamp |
| `r.formType` | String | `10-Q` |
| `r.periodOfReport` | String | Reporting-period end date |
| `r.primaryDocumentUrl` | String | SEC primary document URL |
| `r.linkToTxt` | String | SEC raw text URL |
| `r.linkToHtml` | String | SEC filing index URL |
| `r.market_session` | String | Timing bucket from ingest |
| `r.returns_schedule` | String (JSON) | Scheduled return-window metadata from ingest |

### ExtractedSectionContent Fields

| Field | Type | Description |
|-------|------|-------------|
| `s.id` | String | Content node ID |
| `s.section_name` | String | Canonical section key (no-space format) |
| `s.content` | String | Section text content |
| `s.content_length` | String | Precomputed content length (comma-formatted; use `size(s.content)` in queries for integer) |

### FinancialStatementContent Fields

| Field | Type | Description |

### FilingTextContent Fields

| Field | Type | Description |
|-------|------|-------------|
| `f.id` | String | Content node ID |

### ExhibitContent Fields

| Field | Type | Description |
|-------|------|-------------|

## Relationship Context

- `(:Report)-[:PRIMARY_FILER]->(:Company)` provides the filing/company binding.
- `(:Report)-[:HAS_SECTION]->(:ExtractedSectionContent)` provides named section text.
- `(:Report)-[:HAS_FINANCIAL_STATEMENT]->(:FinancialStatementContent)` provides structured statement payloads.
- `(:Report)-[:HAS_FILING_TEXT]->(:FilingTextContent)` provides raw full-document fallback text.
- `(:Report)-[:HAS_EXHIBIT]->(:ExhibitContent)` provides exhibit text by exhibit number.

Use the graph edges to navigate available content layers. Which layer is relevant is type-specific.

## Content Fetch Order

### Step 1: Load Report Inventory

Run query 5F first. It is the source-specific inventory and metadata payload for the filing.

### Step 2: Fetch the Relevant Section Layer

- Use 5B when the type wants the canonical MD&A section.
- Use 5D to inspect available section names.
- Use 5I to fetch any specific section named by 5D.

### Step 3: Fetch Other Content Layers as Needed

- Use 5C for financial statement payloads.
- Use 5H for exhibit text by exhibit number.
- Use 5G only when the type explicitly needs raw filing text or no higher-level layer is usable.

### Step 4: Bound Raw Filing Text

`FilingTextContent` is the full filing document, not a curated section extract. Bound or slice it before model input; do not treat it like ordinary section text.

### Step 5: Empty Result Is Valid

If the relevant layer is absent or contains no content useful to the extraction type, return an empty result rather than forcing a match.

## Period Identification

10-Qs often mix explicit fiscal periods with relative phrases anchored to the filing date.

| Source Text | Likely period interpretation |
|-------------|------------------------------|
| `quarter ended June 29, 2025` | Specific quarter from source text |
| `fiscal 2025` | Annual fiscal period |
| `remainder of fiscal 2025` | Remaining annual window |
| `second half of fiscal 2025` | Half-year window |
| `next twelve months` | Relative annual window from filing date |
| `by 2027` | Long-range target year |

### given_date

Always use `r.created` as the public disclosure timestamp.

### source_key

Use the content-layer identifier that actually produced the evidence:

- Section content: `s.section_name` unless the extraction type defines a normalized alias
- Financial statements: `fs.statement_type`
- Filing text: `"filing_text"`
- Exhibits: `e.exhibit_number`

`r.periodOfReport` is the reporting-period end date, not the disclosure timestamp.

## Empty Content Handling

| Scenario | Action |
|----------|--------|
| Canonical MD&A missing | Use 5D/5I or another content layer |
| Requested section missing | Try another relevant layer or return empty result |
| Only filing text exists | Bound the raw text before extraction |
| All layers absent | Return an empty-content result for the requested source |

## Source Interpretation Traps

- `section_name` values use compact no-space strings.
- Internal MD&A sub-headings are embedded inside one section blob; they are not separate graph nodes.
- `FinancialStatementContent.value` and `r.returns_schedule` are JSON strings.
- `r.periodOfReport` is the period end, not the point-in-time publication date.
- Most exhibits are contracts or schedules rather than narrative summaries; exhibit relevance is type-specific.
- Raw filing text is much larger than section content and should be treated as a last-resort content layer.
