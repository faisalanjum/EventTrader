# 8-K Extraction Profile

Per-source extraction rules for 8-K filings. Loaded by the extraction agent when `ASSET = 8k`.

## Asset Metadata
- sections: full
- label: 8-K
- neo4j_label: Report

## Data Structure

8-K content comes from multiple layers. Use the content inventory query (4G) first to determine what exists.

| Layer | Node Label | Relationship | Priority |
|-------|-----------|--------------|----------|
| **Exhibit (EX-99.x)** | ExhibitContent | `Report-[:HAS_EXHIBIT]->` | Primary for Items 2.02, 7.01, 8.01 — press releases, presentations |
| **Exhibit (EX-10.x)** | ExhibitContent | `Report-[:HAS_EXHIBIT]->` | Primary for Item 1.01 — material contracts/agreements (51%) |
| **Section** | ExtractedSectionContent | `Report-[:HAS_SECTION]->` | Secondary — item text (primary for Items 5.07, 2.06) |
| **Financial Statement** | FinancialStatementContent | `Report-[:HAS_FINANCIAL_STATEMENT]->` | Structured JSON — not observed for 8-K, check 4G |
| **Filing Text** | FilingTextContent | `Report-[:HAS_FILING_TEXT]->` | Fallback — full filing, avg 690KB |

### Report Metadata (from Report node)

| Field | Type | Example |
|-------|------|---------|
| `r.accessionNo` | String | `0000320193-25-000008` |
| `r.id` | String | Same as accessionNo |
| `r.created` | String (ISO) | `2025-01-30T16:30:00Z` |
| `r.formType` | String | `8-K` |
| `r.items` | String (JSON) | `Item 2.02, Item 9.01` |
| `r.market_session` | String | `after_hours`, `pre_market`, `regular` |
| `r.periodOfReport` | String | `2025-01-25` |

### ExhibitContent Fields

| Field | Type | Description |
|-------|------|-------------|
| `e.exhibit_number` | String | `EX-99.1`, `EX-99.2` (press releases/presentations), `EX-10.1`, `EX-10.2` (contracts). Dirty variants exist (`EX-99.01`, `EX-99.-1`) |
| `e.content` | String | Full exhibit text. Press releases avg 10-50KB. |

### ExtractedSectionContent Fields

| Field | Type | Description |
|-------|------|-------------|
| `s.section_name` | String | Canonical section key (see table below) |
| `s.content` | String | Section text content |

### FinancialStatementContent Fields

| Field | Type | Description |
|-------|------|-------------|
| `fs.statement_type` | String | `BalanceSheets`, `StatementsOfIncome`, `StatementsOfCashFlows`, `StatementsOfShareholdersEquity` |
| `fs.value` | String (JSON) | Structured financial data — needs JSON parsing, not direct field read |

### FilingTextContent Fields

| Field | Type | Description |
|-------|------|-------------|
| `f.content` | String | Full filing text. Avg 690KB for 8-K — scan with keyword windows, not full LLM processing |
| `f.form_type` | String | `8-K` |

### Amendments (8-K/A)

Amended 8-K filings have `formType: '8-K/A'` (514 in database) with `r.isAmendment = true`. Queries 4A/4B/4H filter on `formType: '8-K'` and will **not** match amendments. To include amendments, use 4H with adjusted formType or add a separate query. An amendment supersedes the original filing for the same items.

## Content Fetch Order

Fetch content in this priority order. Primary layer (EX-99.1) takes precedence; check secondary layers for additional metrics not covered by the primary.

### Step 1: Check Content Inventory

Run query 4G to see what exists:
```
exhibits, sections, financial_stmts, filing_text_count
```

### Step 2: Fetch EX-99.1 (Primary)

Query 4C with `source_key = 'EX-99.1'`. This is the press release — the richest source for 8-K content.

If EX-99.1 is missing, check EX-99.2 (presentation). Discovery query 4D lists all exhibits.

**Dirty exhibit numbers**: Some filings use non-standard exhibit numbering (`EX-99.01`, `EX-99.-1`). If exact match on `EX-99.1` fails, use query 4D to discover actual exhibit numbers and match the closest.

### Step 3: Fetch Section Text (Secondary)

Query 4E for the relevant section. Key `section_name` values for 8-K (no-space format):

| section_name | 8-K Item | Content |
|-------------|----------|---------|
| `ResultsofOperationsandFinancialCondition` | Item 2.02 | Earnings announcement text |
| `RegulationFDDisclosure` | Item 7.01 | Mid-quarter updates, pre-announcements |
| `OtherEvents` | Item 8.01 | Material events, strategic updates |
| `FinancialStatementsandExhibits` | Item 9.01 | Exhibit index (rarely contains extractable content) |
| `EntryintoaMaterialDefinitiveAgreement` | Item 1.01 | Material contracts |
| `BankruptcyorReceivership` | Item 1.03 | Bankruptcy filings |
| `CompletionofAcquisitionorDispositionofAssets` | Item 2.01 | M&A transactions |
| `CreationofaDirectFinancialObligationoranObligationunderanOff-BalanceSheetArrangementofaRegistrant` | Item 2.03 | Debt obligations |
| `MaterialImpairments` | Item 2.06 | Asset impairments |
| `NoticeofDelistingorFailuretoSatisfyaContinuedListingRuleorStandard;TransferofListing` | Item 3.01 | Delisting notices |
| `DepartureofDirectorsorCertainOfficers;ElectionofDirectors;AppointmentofCertainOfficers:CompensatoryArrangementsofCertainOfficers` | Item 5.02 | Executive changes |
| `SubmissionofMatterstoaVoteofSecurityHolders` | Item 5.07 | Shareholder votes |

### Step 4: Filing Text (Fallback)

Query 4F. Only if exhibit and section both failed or yielded zero extractable content. This is the full filing text (~690KB avg) — scan with keyword windows, not full LLM processing.

### Content Strategy by Item

Where content lives varies by item type. Intersection files (slot 4) use this to determine fetch priority per item:

```
EXHIBIT-FIRST (check EX-99.x / EX-10.x first):
  Item 2.02 (94% in exhibit), Item 7.01 (85%)

CHECK BOTH (data may be in either):
  Item 1.01 (51% in EX-10.x), Item 8.01 (64% exhibit), Item 5.02 (57% exhibit)

SECTION-FIRST (data usually in section itself):
  Item 5.07 (>99% section), Item 2.06 (98% section)
```

## Period Identification

### Common Patterns in 8-K Press Releases

| Source Text | period_type | Derivation |
|-------------|-------------|------------|
| "For the quarter ending March 2025" | quarter | Use filing date + FYE to determine fiscal quarter |
| "For fiscal year 2025" | annual | fiscal_year from source |
| "For the second half of fiscal 2025" | half | fiscal_year from source |
| "For the next twelve months" | annual | From filing date |
| Table header: "Q2 FY2025 Outlook" | quarter | Direct from table |

### given_date

Always `r.created` (the filing date). This is the point-in-time stamp for when the content became public.

### source_key

The exhibit or item key that contained the content:
- `"EX-99.1"` — most common (press release)
- `"EX-99.2"` — presentation/supplemental
- `"EX-10.1"`, `"EX-10.2"` — material contracts/agreements
- `"Item 2.02"` — section text (earnings)
- `"Item 7.01"` — Reg FD section text
- `"Item 1.01"` through `"Item 8.01"` — other item section text (use the item code that contained the content)

## Basis, Segment, Quality Filters

See core-contract.md S6 (Basis Rules), S7 (Segment Rules), S13 (Quality Filters).

### 8-K Trap: Table Columns

Tables often have GAAP and non-GAAP columns. Extract BOTH — each gets its own extraction node with the appropriate `basis_norm`. Do not skip the GAAP column because the non-GAAP is "more useful."

### 8-K Quality Addition: Safe-Harbor Proximity

Filter disclaimer-only blocks, but do not blindly drop adjacent lines if they contain concrete forward-looking numbers/periods.

## Empty Content Handling

| Scenario | Action |
|----------|--------|
| EX-99.1 missing, section content exists | Process section content — valid extraction |
| EX-99.1 exists but is empty (`strip()==""`) | Return `EMPTY_CONTENT\|8k\|EX-99.1`, try section |
| No exhibit AND no section content | Try filing text (fallback) |
| Filing text also empty | Return `EMPTY_CONTENT\|8k\|full` |
