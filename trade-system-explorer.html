Priority Order

B1. Maximize profit

Enter fast after 8-K (72% win at 5-min, 54% at 60-min). Recalibrate on transcript/news evidence.

B2. Cut losses fast

ATR-clamped stops. Dollar-risk sizing. Broker-native stop enforcement. Kill-switch at account level.

B3. Let winners run

Average winner +12.1%. A 5% cap destroys 80% of returns. Default action = HOLD. Wide thresholds.

B4. Text recalibration

Reassess on transcript + whitelisted Benzinga news. Not price noise. 2-3 reassessments max per trade.

B5. Keep it minimal

One daemon. Files on disk. Broker handles real-time stops. Three-layer ownership model.

B6. Paper first

Paper trading validates plumbing. Two independent flags: ACCOUNT_MODE + DATA_MODE.

Explicitly Out
No continuous LLM monitoring No custom per-trade thresholds No portfolio optimizer No re-entry after exit Long-only at launch No trailing stop No competing trade replacement No FAST/CONTESTED profiles

Fail-Safe Chain

LLM fails safe

Reassessment API error, session expired, rate limit

Action: no_change + no_escalation. Hold position. Hard stops remain active. System correct without reassessment.

Daemon fails safe

Pod crash, OOM, network partition

Action: Broker-native stop still protects. Restart reads plan.json, reconciles with IBKR, resumes monitoring.

Everything fails safe

LLM down + daemon crash + Redis down

Action: IBKR stop fires at set price regardless. Worst case = stop loss. No runaway position ever.

Core invariant: There is never a live position without a broker-native stop. If stop placement fails after entry, the daemon flattens immediately. Broker stop is the bottom of the safety stack.

Entry Decision
Gate checks → sizing math → order placement. One conservative edge input used everywhere: edge = low_end(expected_move_range).
Gate — Two Classes of Checks

Fixed Checks from prediction, won't change

  • direction == long  (short logged, not traded)
  • confidence ≥ 75
  • edge ≥ 3%  (low end of expected_move_range)

Any fail → skip permanently

Dynamic Checks from market, may change

  • Spread: three-tier policy (trade now / defer / skip)
  • Chase: stock moved > 70% of predicted range? → skip
  • Capital: enough free cash? → skip
  • Positions: under MAX_POSITIONS (2)? → skip

Only spread triggers DEFER. All other dynamic = permanent skip.

Three-Tier Spread Policy
soft_max = min(1.0%, 25% × edge) hard_max = min(2.0%, 40% × edge) Example: prediction [5%, 8%] → edge = 5% soft_max = min(1.0%, 1.25%) = 1.0% hard_max = min(2.0%, 2.0%) = 2.0%
TierConditionAction
Trade Nowspread ≤ soft_maxEnter immediately with marketable limit
Defersoft_max < spread ≤ hard_maxRetry until next open + 10 min. Re-run ALL checks each retry.
Skipspread > hard_maxSkip permanently. Friction exceeds opportunity.
Sizing — Dollar-Risk Formula
0. dollar_risk = account_equity × RISK_PER_TRADE_PCT / 100 1. entry_price = current ask (+ buffer in extended hours) 2. stop_distance = clamp(2.0 × ATR_14, 3% × price, 15% × price) 3. per_share_risk = stop_distance + $0.30 execution buffer 4. shares = floor(dollar_risk / per_share_risk) 5. stop_price = entry_price - stop_distance
Example: $50K account, 2% risk → dollar_risk=$1,000. AAPL ask $185.20, ATR=$4.50 → stop=$9.00 → per_share_risk=$9.30 → shares=107 → max planned loss=$995.
Small account: $2.5K account, 2% risk → dollar_risk=$50 → shares=5 → max planned loss=$46.50.
Note: actual loss may exceed planned if stock gaps past stop (earnings/extended-hours risk).
Order Placement

Preferred: 2-Leg Bracket

Parent: BUY shares @ marketable limit

Child: SELL shares @ STOP stop_price

Placed atomically. If entry fills, stop is automatically active. No take-profit leg — "let winners run."

Fallback: Fail-Closed

1. Place entry → wait for fill

2. Immediately place stop for filled qty

3. If stop NOT confirmed → FLATTEN immediately

Partial fill: stop matches FILLED shares. Cancel unfilled remainder.

Data Mode Determines Monitoring Style

DATA_MODE=delayed launch

Poll IBKR every 30s. Check thresholds each cycle. 15-min delayed prices. Good for plumbing test.

DATA_MODE=realtime goal

IBKR streaming quotes. Callback on each update. Sub-second threshold detection. Required for go-live.

Safety Net
Singleton lock, startup recovery, reconciliation, kill switch. Every failure path either resolves safely or escalates to manual intervention.
Daemon Singleton (Redis Lock)
startup: redis.set("trade_daemon:lock", pod_id, nx=True, ex=60) running: every 30s → atomic compare-and-renew (Lua script) shutdown: atomic compare-and-delete (Lua script) Redis down at startup → refuse to start (safe: no duplicate risk) Redis down during operation → monitoring continues, new trades stop
Startup Recovery (5 Steps)
1
Acquire Redis lock — prevents duplicate daemons
2
Scan ALL trade/plan.json files — reconcile each by status (see Level 4)
3
Crash safety: DEFERRED → SKIPPED — if both plan.json (deferred) AND skipped.json exist, skipped wins
4
Scan for untraded predictions — result.json without plan.json or skipped.json → process normally
5
Resume normal operation — Redis BRPOP + monitoring for active trades
IBKR Reconciliation (every 5 minutes)
1
Position check — IBKR position exists? Size matches plan.json? Missing = stop fired.
2
Stop order status — Still active? Cancelled/expired → re-place immediately. Repair fails → FLATTEN.
3
Orphan detection — IBKR position with no matching plan.json → log CRITICAL. Do NOT auto-close.
4
Timestamp update — Update last_reconciled_at in plan.json
Account Kill-Switch
If account_equity < starting_equity × (1 - 15%): close ALL positions with market orders. Daemon enters HALTED mode — no new trades, only reconciliation. Requires manual restart.

starting_equity captured once at first startup, persisted to earnings-analysis/trade_daemon_state.json. Does NOT reset on restart.
